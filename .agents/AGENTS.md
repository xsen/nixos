# Правила проекта (NixOS xsen)

Этот файл содержит контекст репозитория и специфичные правила поведения агентов для NixOS-конфигурации.

## 1. Специфика окружения и архитектуры
* **Хост:** `nix-desktop`, **Пользователь:** `evgeny`, **Система:** `x86_64-linux`.
* **Связь с системой:** Репозиторий линкуется в `~/.nix-config` через `./install.sh`. Переменная `NH_FLAKE` указывает на `~/.nix-config` (задается в `home/home.nix`).
* **Архитектурная структура:**
  - `overlays.nix` — Оверлеи для кастомных пакетов (evelens, antigravity-cli, фиксы для discord и koreader).
  - `hosts/nix-desktop/configuration.nix` — Системная конфигурация (boot, nvidia, services, networking).
  - `hosts/nix-desktop/home-manager.nix` — Точка входа в home-manager, подключает `home/home.nix`.
  - `home/home.nix` — Главный home-manager модуль (обои, скрипты, сессия).
  - `home/packages.nix` — Пользовательские пакеты (ghostty, kitty, vim, btop и др.).
  - `home/hypr/` — Конфигурация Hyprland на Lua (`hyprland.lua`), вспомогательные конфиги (`hyprlock`, `hypridle`, `hyprpaper`), разворачиваемые через `xdg.configFile`.
  - `modules/` — Системные модули (подключаются в `configuration.nix`):
    * `modules/packages.nix` — Системные пакеты + Hyprland, Thunar.
    * `modules/dev.nix` — Среда разработки (Docker, Nginx, Ollama, Node.js, PHP, IDE).
    * `modules/dev-hosts.nix` — Виртуальные хосты для Nginx.
    * `modules/games.nix` — Игровые утилиты и Steam.
    * `modules/nix-ld.nix` — Активация nix-ld для запуска unpatched ELF бинарников.
    * `modules/polkit.nix` — Правила авторизации Polkit.

## 2. Специфика Nix Flakes и Git
* Любые новые `.nix` или `.conf` файлы должны быть добавлены в git-индекс с помощью команды `git add -N <file>` (или `git add <file>`). Если файл не отслеживается Git, сборщик Flakes проигнорирует его, что приведет к ошибкам импорта.

## 3. Команды сборки и тестирования (для агента)
Агент **не должен самостоятельно запускать применение конфигурации** (`nh-os`, `nh-home`, `nh-all`), так как это требует ввода пароля `sudo` и приведет к зависанию терминала в песочнице.

* **Тестирование изменений (выполняет агент)**:
  Перед сдачей задачи агент должен убедиться, что конфигурация собирается без ошибок. Для этого используй:
  * Быстрая проверка синтаксиса: `nix-instantiate --parse <file>`
  * Полная валидация флейка: `nix flake check`
  * Тестовая сборка системы (в локальную папку `./result` без применения):
    `nix build .#nixosConfigurations.nix-desktop.config.system.build.toplevel`
  * Тестовая сборка home-manager (без применения):
    `nix build .#homeConfigurations.evgeny.activationPackage`

* **Применение изменений (выполняет пользователь)**:
  После успешной тестовой сборки агент должен предложить пользователю применить изменения, выведя соответствующую команду:
  * `nh-os` — сборка и применение системной конфигурации.
  * `nh-home` — сборка и применение домашней среды (home-manager).
  * `nh-all` — последовательная сборка системы и домашней среды.


## 4. Форматирование кода
* Любой измененный или созданный Nix-файл должен быть отформатирован перед коммитом с помощью:
  `nixfmt <file.nix>`

## 5. Важные технические особенности
* **Принцип разделения конфигураций:** Не пиши конфигурационные файлы прикладных программ (Hyprland, Waybar, Rofi и т.д.) целиком внутри Nix-кода Home Manager. Выноси их в отдельные raw-файлы (Lua, JSON, CSS) в соответствующие директории (например, `home/hypr/`, `home/waybar/`) и разворачивай через `xdg.configFile`. Это необходимо для сохранения поддержки LSP, автодополнения и форматирования в текстовых редакторах.
* **Драйверы Nvidia:** Проприетарные драйверы с фиксированной версией настраиваются через функцию `mkDriver` в `hosts/nix-desktop/configuration.nix`. Для обновлений используй скрипт авто-обновления Nvidia (если доступен скилл `update-nvidia`).
* **Локализация:** Системный язык интерфейса — русский (`ru_RU.UTF-8`), раскладка клавиатуры `us,ru-pro` (кастомная раскладка с символами США на AltGr) с переключением по CapsLock.
* **Секреты:** Не хардкодь пароли, токены или ключи в Nix-файлах. При необходимости настройки секретов спроси пользователя.
