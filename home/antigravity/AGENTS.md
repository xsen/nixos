## Браузерная автоматизация и MCP
- **Приоритетный браузер**: Для любых задач по автоматизации, тестированию или отладке веб-страниц по умолчанию используй инструменты **Chrome DevTools** (плагин `chrome-devtools-mcp`). Инструменты Playwright используй только при явном указании пользователя.
- **Особенности окружения (NixOS)**: Запуск Chrome DevTools MCP и Playwright MCP осуществляется через специальные скрипты-обертки, расположенные в каталоге плагина.

### Режимы запуска и Переменные окружения
При запуске браузера учитывай следующие переменные окружения, которые пользователь может передавать в CLI:

#### Для Chrome DevTools (приоритетный):
- `AGY_CHROME_HEADLESS` (по умолчанию `true`): если установлено в `false`, браузер запустится с отображением графического окна (headed).
- `AGY_CHROME_ISOLATED` (по умолчанию `true`): если установлено в `false`, используется постоянный профиль пользователя в `~/.config/chrome-mcp-profile` (сохраняет сессии и авторизации, например для amoCRM). По умолчанию (`true`) используется временный изолированный профиль.

#### Для Playwright (резервный):
- `AGY_PLAYWRIGHT_HEADLESS` (по умолчанию `true`): если `false`, запускает headed-режим.
- `AGY_PLAYWRIGHT_ISOLATED` (по умолчанию `true`): если `false`, использует постоянный профиль в `~/.config/playwright-mcp-profile`.

*Пример запуска для ручной авторизации на сайтах (headed + non-isolated):*
`AGY_CHROME_HEADLESS=false AGY_CHROME_ISOLATED=false agy`

### Динамическое управление режимами по запросу пользователя
Если пользователь просит запустить браузер в специфическом режиме в процессе диалога (например, *"проверь через хром не в изолированном режиме с показом окна"*):

1. **Для Chrome DevTools (предпочтительно)**:
   - Не полагайся на дефолтный запуск MCP. Вместо этого принудительно запусти браузер самостоятельно в фоновом процессе через `run_command` с нужными флагами:
     - Оконный режим (headed): без флага `--headless`
     - Неизолированный режим (non-isolated): `--user-data-dir=/home/evgeny/.config/chrome-mcp-profile`
     - Порт отладки: `--remote-debugging-port=9222`
   - После этого вызывай MCP-инструменты `chrome-devtools`. Локальный скрипт плагина автоматически подключится к уже запущенному тобой инстансу Chrome на порту 9222.
   
2. **Для Playwright / Сложных сценариев**:
   - Напиши временный Node.js скрипт автоматизации в директорию `scratch` (используя пакет `playwright` из кэша `/home/evgeny/.npm/_npx/9833c18b2d85bc59/node_modules`), настроив в объекте `chromium.launch` любые параметры по запросу пользователя. Запусти его через `run_command`.
